# `carousel-composer` subagent

**One job:** Given an inventory + reach-results for all 10 shots, produce the generated `<game>-v6.json` spec. Order shots by per-genre conversion psychology. Write captions denylist-clean. Pick gradients aligned to brand. Sub-text variation across shots. Validate the whole spec.

**Output:** `scripts/store-screenshots/compositions/<game>-v6.json` (overwrites the hand-authored v7 version once approved)

**Charter scope:** Composes specs from upstream subagent outputs. Never executes capture (that's `capture.mjs`'s job). Never proposes new hooks (that's `hook-designer`'s job).

---

## Trigger conditions

After `state-reacher` produces reach-results for all 10 shots in a game's planned carousel. The brain dispatches `carousel-composer` to assemble the v8-generated spec.

## Inputs

- `game_name`, `game_inventory_path`, `reach_results_dir` (`brain/aso/reach-results/<game>/`)
- `genre_canon` — per-genre hook ordering from `brain/aso/README.md` (puzzle / arcade / social)
- `competitor_research` (optional) — most recent `[market-intel]` issue body for the game

## Tools

Read, Write (the v6.json spec only).

**Forbidden:** Bash (no capture.mjs invocation), gh CLI, Edit (no game source).

---

## Hook ordering canon

Reproduced from `brain/aso/README.md` for offline subagent reference:

### Puzzle genre (AP, Two Dots, Threes, Flow Free)
1. **Hard-puzzle / "I want to solve THAT"** — packed mid-expert state
2. **Variety** — 2-3 puzzles at different difficulty tiers showing range
3. **Streak / "I'll come back daily"** — daily challenge with multi-day streak
4. **Depth / "this won't run out"** — levels grid scrolled, star completion visible
5. **Personalization** — themes shown on actual gameplay (not just menus)
6. **Brand** — clean menu hero, last position

### Arcade-explosion (BX, Block Blast, Royal Match, Toon Blast)
1. **Combo-dopamine / "WOAH that combo"** — mid-explosion ×N with score popup
2. **Mastery** — ×8 MEGA / ULTRA / multiplier-ceiling shot
3. **Status** — best-score celebration with crown
4. **Progression** — adventure map with progress + completion stars
5. **Tension** — revive countdown / IAP-adjacent moment
6. **Brand** — menu hero

### Realtime social (HM, Among Us, Mafia/Werewolf clones)
1. **Friends-now / "play with friends right now"** — full lobby with named players + room code
2. **Roles** — role reveal screen
3. **Action** — night/voting action mid-frame
4. **Stakes** — game-over with victory state visible
5. **Brand** — title with Create Room CTA

For 10-shot carousels, expand each genre's 5-hook canon by adding within-bucket variants (e.g. puzzle → 4 puzzle shots at 4 difficulties for the "hard-puzzle + variety" pair).

## Caption rules

- **Length:** 1-3 lines vertical, 6-12 chars per line ideal
- **Tone:** declarative, present-tense, second-person implicit ("Tap. Flip. Solve." not "Players will tap and flip")
- **Variation:** no two captions in the same carousel use the same first word
- **Denylist:** every caption + sub regex-checked against the denylist before write (App Store / Play compliance — see `capture.mjs` `CAPTION_DENYLIST`)
- **Sub-text:** 4-8 words, expands on caption with concrete benefit ("Mid-expert puzzles get dense fast. Plan every flip.")
- **Numeric claims:** every numeric claim (`400+`, `90 stages`, `Streak 7`) must have evidence in `scripts/store-screenshots/claims/<game>.json` (rule 7)

## Gradient rules

- 4-stop linear gradient per shot
- Aligned to game's brand palette but varied per shot for carousel rhythm
- Hero shot uses the brand's most saturated palette
- Theme-variant shots use accent colors matching the in-screen theme (ocean shot → teal gradient)

## Validation before write

The spec must pass these checks before writing:

- [ ] 10 compositions present (or `target_shot_count` if game spec overrides)
- [ ] All compositions have unique `id` matching `^\d{2}-[a-z0-9-]+$`
- [ ] Every caption + sub passes the denylist regex
- [ ] Every composition has `provenance: "playwright-capture"`
- [ ] Hero shot (#1) is NOT a menu / title / splash / tutorial unless `shot_1_menu_justification` is set in the spec
- [ ] No two adjacent shots use the same gradient palette (carousel rhythm)
- [ ] Numeric claims in captions all have entries in `claims/<game>.json` (or claim-substantiation file is generated alongside)
- [ ] Every composition's `_source_reach_result` points to an existing file under `brain/aso/reach-results/<game>/`

If any check fails, do NOT write the spec. Output a validation-failed report.

## Operating procedure

1. **Read inventory** — pull brand, themes, "addictive hypothesis", density estimates.
2. **Read genre canon** from this README's "Hook ordering canon" section.
3. **Read all reach-results** for this game's planned 10 shots.
4. **Compose compositions array:**
   - Map each reach-result to a composition object
   - Set `actions` from reach-result's Path 1/3 sequence
   - Set `assert_screen` from inventory's renderer entry point
   - Pick caption + sub aligned to the shot's hook position
   - Pick gradient from brand palette + carousel rhythm rule
5. **Order by genre canon:** hero shot first, then variety, then retention, then depth, then personalization, then brand.
6. **Validate** against the checklist above.
7. **Write spec** to `scripts/store-screenshots/compositions/<game>-v6.json`.
8. **Generate `claims/<game>.json`** if numeric claims are present and the file doesn't exist (or update if claims changed).

## Anti-patterns

- **Hand-editing the spec after generation.** v8 specs are generated. Re-run the composer to regenerate. If the composer keeps producing wrong output, fix the composer (this README), not the spec.
- **Hero shot is a menu.** Never (unless explicit `shot_1_menu_justification`). Aspiration leads, brand follows.
- **Captions all start with the same word.** Carousel feels monotonous. Vary first words.
- **Skipping denylist check.** App Store rejection risk.
- **Inventing reach-results.** Every composition must trace to a real reach-result file.
- **Skipping validation before write.** A spec that validates is a spec that captures cleanly.

## Self-cleanup before exit

- Confirm only one file written: `scripts/store-screenshots/compositions/<game>-v6.json`
- Append one-line summary: `composed: <game> → 10 shots, hook-order matches genre canon, denylist-clean, validation passed`
